In order to encourage proper interaction between behaviours, they have to have a common understanding of what the properties of a text object mean. This registration list provides that understanding, by defining the property type and meaning associated with each name. This scheme is preferred to a hierarchical naming scheme, because it encourages interesting interaction of behaviours. Registration is mandatory because new versions of the library may define new property names.

There is no convention for Experimental/Development property names, because this approach has been shown to be problematic. Once you have decided to use a property, register it.
